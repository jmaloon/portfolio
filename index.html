Colour Palette

Colour is a powerful tool for building cohesive digital experiences. For this website I've implemented a colour system that has deep roots that touch all the colours presented on this site. The hue of the accent colours are also present in the text and background colours. Using colours with shared hues creates a sense of continuity that would not exists without a colour system. This website uses a single hue value to determine the entire colour palette.

Below you can manually change the hue with the range input or choose a preset colour with one of the buttons.

The colour system is built with CSS Variables and hsla colour definitions.

Using hsla colour definitions significantly simplified the dynamic colour functionality problem of this website. Since hsla functional notation accepts a hue value, all I had to do was create a CSS Variable that represents the dynamic hue value and update said variable using JavaScript.

Using CSS Variables was an easy decision as it allows me to define values in one location and use the variables throughout the stylesheet(s). This makes it super easy to create a "Dark Mode" as I can redefine the colour definitions in one location and...voila!

Font Loading

This website uses the Flash Of Unstyled Text (FOUT) loading strategy to handle font loading. The FOUT strategy renders a system font until the web font is loaded and ready to be presented. This functionality can be achieved with the browser natively using the font-display: swap property, however I chose to implement using JavaScript for better support and more control.

Handling font loading is only part of the story. While the FOUT strategy doesn't block page rendering, it may lead to dramatic layout shifts when the system font is replaced by the web font. The layout shifts cannot be totally avoided but they can be minimized by managing the differences in type kerning, leading, and tracking.

The button below will change the font for the page. Click and observe how the content shifts when the font changes!
